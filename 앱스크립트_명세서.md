# 메디스파크 랜딩페이지 Apps Script 명세서

## 1. 목적

랜딩페이지에서 수집한 고객 DB를 Google Apps Script 웹앱으로 전송하고, Google Sheets에 저장한다.

본 명세서는 아래 흐름을 기준으로 한다.

1. 사용자가 `상담하기` 버튼 클릭
2. 이름과 연락처 입력
3. 폼 제출
4. Apps Script 웹앱으로 데이터 전송
5. Google Sheets 저장
6. 저장 직후 `010-6689-2348` 전화 연결

## 2. 고정 운영값

- 프로젝트명: `브레인시티 메디스파크`
- 대표번호: `010-6689-2348`
- 개발자정보: `이신우`

## 3. 수집 필드 정의

### 3-1. 필수 필드

- `name`: 고객명
- `phone`: 고객 연락처
- `developer`: 개발자정보

### 3-2. 권장 추가 필드

- `inquiryType`: 문의유형
- `unitType`: 관심평형
- `visitDateTime`: 방문희망일시
- `consent`: 개인정보동의 여부
- `pageUrl`: 유입 페이지 주소
- `device`: 사용자 기기
- `createdAt`: 접수 시각

## 4. 전송 규격

외부 랜딩페이지에서 Google Apps Script 웹앱으로 전송할 때는 `POST` 방식의 HTML form submit을 권장한다.

권장 이유:

- 외부 도메인 환경에서 구현이 단순함
- CORS 이슈를 우회하기 쉬움
- 숨김 iframe과 함께 쓰면 페이지 이탈 없이 저장 가능

### 4-1. 요청 방식

- Method: `POST`
- Content-Type: `application/x-www-form-urlencoded`
- Target: 배포된 Apps Script 웹앱 URL

### 4-2. 요청 파라미터 예시

```text
name=홍길동
phone=01012345678
developer=이신우
inquiryType=phone_call
unitType=84A
visitDateTime=
consent=Y
pageUrl=https://example.com/landing
device=mobile
createdAt=2026-06-27T15:40:00+09:00
```

## 5. Apps Script 요구사항

### 5-1. 스프레드시트 저장 요구사항

Apps Script는 `doPost(e)`에서 아래 작업을 수행해야 한다.

1. 요청값 파싱
2. 필수값 검증
3. 스프레드시트 열기
4. 시트 마지막 행에 append
5. 결과 반환

### 5-2. 필수 검증 조건

- `name`이 비어 있으면 저장하지 않음
- `phone`이 비어 있으면 저장하지 않음
- `developer`가 비어 있으면 기본값 `이신우` 사용

### 5-3. 연락처 정규화 권장

전화번호는 저장 전 숫자만 남기도록 정리하는 것이 좋다.

예:

- 입력값: `010-6689-2348`
- 저장값: `01066892348`

## 6. Google Sheets 구조

시트명 권장:

- `leads`

헤더 순서 권장:

1. 접수일시
2. 이름
3. 연락처
4. 개발자정보
5. 문의유형
6. 관심평형
7. 방문희망일시
8. 개인정보동의
9. 유입페이지
10. 사용자기기
11. 원본IP대체값 또는 비고

## 7. 랜딩페이지 동작 요구사항

### 7-1. 상담 팝업

- 필드 수는 `이름`, `연락처`만 노출
- 개발자정보 `이신우`는 숨김 필드로 전송
- 개인정보 동의 체크는 필수

### 7-2. 저장 후 행동

저장 요청을 보낸 뒤 아래 순서로 동작시킨다.

1. 중복 제출 방지
2. 숨김 iframe으로 폼 submit
3. 0.6초~1.0초 후 전화 연결
4. 실패 대비 문구 노출

### 7-3. 실패 대비 문구

- 잠시 후 전화 연결됩니다.
- 연결이 되지 않으면 `010-6689-2348`로 직접 전화 부탁드립니다.

## 8. 사용자 거부감 완화 문구

상담 팝업에는 아래 문구를 권장한다.

- 성함과 연락처만 남겨주시면 빠르게 상담 연결을 도와드립니다.
- 상담 요청 확인을 위한 최소 정보만 받고 있습니다.
- 남겨주신 정보는 요청하신 상담 진행 외 다른 용도로 사용되지 않습니다.

## 9. 배포 체크리스트

### 9-1. Apps Script

- 스프레드시트 ID 설정
- 시트명 확인
- 웹앱 배포
- 접근 권한을 `Anyone` 또는 운영 정책에 맞게 설정

### 9-2. 랜딩페이지

- 웹앱 URL 반영
- 숨김 필드 `developer=이신우` 반영
- 상담 팝업 연결 확인
- 개인정보 동의 체크 확인
- 모바일 전화 연결 테스트 확인

## 10. 최종 구현 권장 방식

실무에서는 아래 조합을 권장한다.

- 프론트: HTML + CSS + JavaScript
- 저장: Google Apps Script 웹앱
- DB: Google Sheets
- 전화 연결: `tel:01066892348`

이 구조는 구현이 단순하고, 수정이 쉬우며, 초기 운영 속도가 빠르다.
